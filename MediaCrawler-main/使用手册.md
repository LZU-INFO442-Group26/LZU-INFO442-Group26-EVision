
## 2. 新项目/新关键词时只改哪里（重点）

每次爬新的项目，主要改这几处：

- `--platform`：平台代码，见第 3 节
- `--keywords`：关键词，例如 `新能源汽车`、`百年传奇`
- `--save_data_path`：保存路径

长期 SQLite 路径，适合长期累积、去重，并作为后续 Excel 导出的唯一数据来源：

```text
datas/MediaCrawler-data/<platform>
```

Excel 不再通过爬虫直接生成，而是从已爬好的 DB 导出：

```text
datas/MediaCrawler-data/<platform>/<platform>_export.xlsx
```

新项目备注：复制命令时，主要改带 `【】` 的内容：

```bash
--keywords "【你的关键词】"
--save_data_path "datas/MediaCrawler-data/【platform】"
```

说明：统一先写入 SQLite DB，再按第 6 节从已有 DB 导出 Excel。不要让爬虫阶段直接生成 Excel。

推荐顺序：

1. 初始化建表
2. 跑爬虫写入 DB
3. 由 DB 导出 Excel

## 4. 所有平台的通用命令模板

### 4.1 初始化建表

把 `【platform】` 和路径里的 `【platform】` 改成平台代码。

```bash
cd "/Users/wangch/coding/Crawler/MediaCrawler-main"
uv run main.py \
  --platform 【platform】 \
  --init_db sqlite \
  --save_data_path "datas/MediaCrawler-data/【platform】"
```

### 4.2 爬虫写入 SQLite

```bash
cd "/Users/wangch/coding/Crawler/MediaCrawler-main"
uv run main.py \
  --platform 【platform】 \
  --lt qrcode \
  --type search \
  --keywords "【关键词】" \
  --save_data_option sqlite \
  --save_data_path "datas/MediaCrawler-data/【platform】"
```

### 4.3 由 DB 导出 Excel

先用 4.2 把爬虫结果写入 SQLite，再用第 6 节的通用导出脚本从已有 DB 导出 Excel。

## 6. 由 DB 导出 Excel（所有平台通用）

这个命令会按第 3 节表格自动识别平台和表，只读 DB，不修改原始数据。

把 `PLATFORM = "xhs"` 改成你要导出的平台代码。

```bash
cd "/Users/wangch/coding/Crawler/MediaCrawler-main"
uv run python - <<'PY'
import sqlite3
from pathlib import Path
import pandas as pd

PLATFORM = "xhs"
CONFIG = {
    "xhs": {"db": "xhs.db", "xlsx": "xhs_export.xlsx", "tables": ["xhs_note", "xhs_note_comment", "xhs_creator"]},
    "dy": {"db": "dy.db", "xlsx": "dy_export.xlsx", "tables": ["douyin_aweme", "douyin_aweme_comment", "dy_creator"]},
    "ks": {"db": "ks.db", "xlsx": "ks_export.xlsx", "tables": ["kuaishou_video", "kuaishou_video_comment"]},
    "bili": {"db": "bili.db", "xlsx": "bili_export.xlsx", "tables": ["bilibili_video", "bilibili_video_comment", "bilibili_up_info", "bilibili_up_dynamic"]},
    "wb": {"db": "wb.db", "xlsx": "wb_export.xlsx", "tables": ["weibo_note", "weibo_note_comment", "weibo_creator"]},
    "tieba": {"db": "tieba.db", "xlsx": "tieba_export.xlsx", "tables": ["tieba_note", "tieba_comment", "tieba_creator"]},
    "zhihu": {"db": "zhihu.db", "xlsx": "zhihu_export.xlsx", "tables": ["zhihu_content", "zhihu_comment", "zhihu_creator"]},
}

cfg = CONFIG[PLATFORM]
base = Path("datas/MediaCrawler-data") / PLATFORM
conn = sqlite3.connect(base / cfg["db"])
with pd.ExcelWriter(base / cfg["xlsx"], engine="openpyxl") as writer:
    existing = {row[0] for row in conn.execute("select name from sqlite_master where type='table'")}
    for table in cfg["tables"]:
        if table in existing:
            pd.read_sql_query(f"select * from {table}", conn).to_excel(writer, sheet_name=table[:31], index=False)
conn.close()
print(f"exported to {base / cfg['xlsx']}")
PY
```

## 9. 常见问题

### 9.1 没有表

如果出现：

```text
sqlite3.OperationalError: no such table: xxx
```

先执行对应平台的初始化建表命令。

### 9.2 登录过期或二维码失败

1. 保持 `--lt qrcode`
2. 重新启动 Chrome CDP
3. 在浏览器里手动登录或完成滑块
4. 再运行爬虫命令

### 9.3 CDP 端口不可访问

```bash
open -na "Google Chrome" --args --remote-debugging-port=9222 --user-data-dir="/tmp/mediacrawler-chrome"
curl -sS http://127.0.0.1:9222/json/version
```

### 9.4 SOCKS 代理报错

```bash
cd "/Users/wangch/coding/Crawler/MediaCrawler-main"
uv pip install "httpx[socks]"
```

## 10. 安全说明

“由 DB 导出 Excel”只执行：

```text
select * from 表名
```

不会执行：

```text
update / delete / drop / alter
```

因此不会修改原始 `.db`，只会新增或覆盖对应的 `*_export.xlsx` 文件。
